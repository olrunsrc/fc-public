  3  The initialisation file has the following format:: 
  4       
  5      [plan1] 
  6      beh1.attr1 = value1 
  7      beh2.attr1 = value2 
  8       
  9      [plan2] 
 10      beh1.attr1 = value3 
 11      beh2.attr2 = value4 
 12       
 13  Such a file specifies two agents to be creates, one using 'plan1', and the 
 14  second using 'plan2'. The first agent is initialised as follows: attribute 
 15  attr1 of behaviour beh1 is set to value1, and attribute attr1 of behaviour 
 16  beh2 is set to value2. The second agent is initialised in a similar way. 
 17   
 18  In more detail, the agent initialisation is specified in a line-wise fashion. 
 19  Comments (starting with a '#') and empty lines are ignored. Every agent block 
 20  in the file start with its plan, given by '[plan]', where 'plan' can be any 
 21  string _without_ spaces. After that an aribtrary number of attributes (one per 
 22  line) can be given for each behaviour of the agent. An attribute is specified 
 23  by 'behaviour.attrribute = value', where 'behaviour' and 'attribute' are 
 24  names of the behaviour and the attribute, respectively, and 'value' is the 
 25  value that the attribute is set to. The following regular expressions 
 26  determine valid names and values:: 
 27       
 28      behaviour and attribute names  [a-zA-Z_][a-zA-Z0-9_]* 
 29      values: 
 30          integer                    \-?[0-9]+ 
 31          float                      \-?(\d*\.\d+|\d+\.)([eE][\+\-]?\d+)? 
 32          boolean                    ([Tt]rue|[Ff]alse) 
 33   
 34  The values are automatically converted into the recognised type. If no type is 
 35  recognised, then they are assigned to the attributes as strings. 
 36  """
