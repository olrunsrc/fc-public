setDriveCollection(self, drivecollection)

source code 

Sets the drive collection of the plan.

The drive collection has to be given in the following format: (type, name, goal, priorities) where:

  • type: string, any of DC, RDC, SDC, SRDC
  • name: string, name of drive collection
  • goal: a goal list as described below _ priorities: a list of comptence priorities as described below

A goal is a sequence of senses and sense-acts, where sense-acts are given by their name as a string, and senses are given by a triple of the form (name, value, predicate), where all elements are given as string. Valid values for predicates are discussed in the documentation of POSH.strict.Sense. If there is no goal, then None can be given instead of the goal, which is treated equivalently to an empty list.

A list of priorities is a sequence of collections of drive elements, like [[drive element 1a, drive element 1b], [drive element 2a], ...], Each drive element is a quadruple given by (name, trigger, triggerable, frequency), where the name is a string, the trigger is - just as a goal - a collection of senses and sense-acts, a triggerable is given by its name, and the frequency is a long integer. If there is no trigger (i.e. the element is always triggered), then None can be given as a trigger.

Parameters:
  • drivecollection (described above) - A structure describing the drive collection.

addActionPattern(self, actionpattern)

source code 

Adds the given action pattern to the plan.

The given action pattern has to be a triple of the form (name, time, action squence), where the name is given by a string, the time is given as a long integer (or None, if no time is specified), and the action sequence is given as a sequence of strings that give the action / sense-act / competence names, or triples of the form (name, value, predicate), where all elements are given as string and the triple describes a sense. Valid values for predicates are discussed in the documentation of POSH.strict.Sense.

Parameters:
  • actionpattern (described above) - A structure describing the action pattern.
Raises:
  • NameError - If there is already an action pattern or competence with the given name in the plan.

addCompetence(self, competence)

source code 

Adds the given competence to the plan.

The competence structure is similar to a drive collection and is described by the quatruple (name, time, goal, priorities). The name is given as string, and the time as a long integer (or None if no time is specified).

A goal is a sequence of senses and sense-acts, where sense-acts are given by their name as a string, and senses are given by a triple of the form (name, value, predicate), where all elements are given as string. Valid values for predicates are discussed in the documentation of POSH.strict.Sense. If either vqlue or predicate are not specified, then None can be given instead of the string. If there is no goal, then None can be given instead of the goal, which is treated equivalently to an empty list.

A list of priorities is a sequence of collections of competence elements, like [[competence element 1a, competence element 1b], [competence element 2a], ...]. Each competence element is a quadruple given by (name, trigger, triggerable, retries), where the name is a string, the trigger is - just as a goal - a collection of senses and sense-acts, a triggerable is given by its name, and the number of retries is is a long integer. If there is no trigger (i.e. the element is always triggered), then None can be given as a trigger.

Parameters:
  • competence (described above) - A structure describing a competence.
Raises:
  • NameError - If there is already an action pattern or competence with the same name in the plan.

build(self, agent)

source code 

Builds the plan and returns the drive collection.

This method operates in several stages:

  1. It is checked if none of the action pattern or competence names are already taken by an action or sense/sense-act in the behaviour library. If a conflict is found, then NameError is raised.
  2. All competence / action pattern objects are created, together with goals and triggers, but their elements are left empty.
  3. The elements of competences and action pattern are created.
  4. The drive collection is built and returned.
Parameters:
  • agent (POSH.strict.Agent) - The agent that uses the plan.
Returns: POSH.strict.DriveCollection
The drive collection as the root of the plan.
Raises:
  • NameError - If clashes in naming of actions / action pattern / competences were found, or if a sense / action / sense-act was not found.

_buildDriveCollection(self, agent, competences, actionpatterns)

source code 

Builds the drive collection and returns it.

This method builds the drive collection, of which the structure has been set by setDriveCollection. Additionally, its assigns the agent a timer, as specified by the drive collection (i.e. a stepped timer, in the case of an SDC drive, and a real-time timer in the case of an SRDC drive). If the timer is a real-time timer, then it is initialised with a loop frequency of 50Hz.

Only drives of type 'SDC' and 'SRDC' are accepted. In any other case a TypeError is raised.

Parameters:
  • agent (POSH.strict.Agent) - The agent that the drive collection is built for.
  • competences (Dictionary, string -> POSH.strict.Competence) - A competence object dictionary.
  • actionpatterns (Dictionary, string -> POSH.strict.ActionPattern) - An action pattern dictionary.
Returns: POSH.strict.DriveCollection
The drive collection.
Raises:
  • TypeError - For drives of types other than SDC or SRDC.

_buildCompetences(self, agent, competences, actionpatterns)

source code 

Completes the competences based on the given competence stubs.

This method modifies the given competence stubs and creates all its elements. These elements can either be other competences, action pattern, or actions. In case of actions, the corresponding POSH.strict.Action objects are created. The actions have to be available in the agent's behaviour dictionary.

Parameters:
  • agent (POSH.strict.Agent) - The agent to build the competences for.
  • competences (Dictionary, string -> POSH.strict.Competence) - The competence stubs, as returned by _buildCompetenceStubs.
  • actionpatterns (Dictionary, string -> POSH.strict.ActionPattern) - The action pattern stubs, as returned by _buildActionPatternStubs

_buildActionPatterns(self, agent, competences, actionpatterns)

source code 

Completes the action pattern based on the given action pattern stubs.

This method modifies the given action pattern stubst and creates all its elements. The elements can either be other competences, actions or senses, where competences are only allowed as the last elements in action patterns. In case of actions / senses, the corresponding POSH.strict.Action / POSH.strict.Sense objects are created. The actions / senses have to be available in the agent's behaviour dictionary.

_getTriggerable(self, agent, name, competences={}, actionpatterns={})

source code 

Returns the action / competence / actionpattern with the given name.

This method looks for the element with the given name and returns it. If the element is an action, then it creates a new POSH.strict.Action object from the agent's behaviour dictionary. Otherwise it just returns the competence or action pattern object.

The method also checks if the the given name is both an action and a competence / action pattern. In that case a NameError is raised.

Parameters:
  • agent (POSH.strict.Agent) - The agent that the element belongs to.
  • name (string) - The name of the element.
  • competences (Dictionary, name -> POSH.strict.Competence) - A competence object dictionary.
  • actionpatterns (Dictionary, name -> POSH.strict.ActionPattern) - An action pattern object dictionary.
Returns: POSH.strict.Action, POSH.strict.Competence or POSH.strict.ActionPattern
The element with the given name.
Raises:
  • NameError - If actions and competences / action pattern have the same name.
