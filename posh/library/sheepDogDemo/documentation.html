Sheep / Dog Demo  (SPOSH version)


This is an example BOD/MASON library built for the strict slip-stack (SPOSH) version of BOD.  SPOSH is a version of POSH written by Jan Drugowitsch, inspired by the earliest version of POSH (previously available only in archaic C++).  In contrast, scheduled POSH includes extra cognitive state, and is therefore slower.  The scheduled POSH version of the basic version of this demo is included in the mason directory.

This library has not been used for any experiments, just for developing BOD/MASON.  It has been described a little by Bryson, Caulfield and Drugowitsch (2006).  See for another example library latch Test, which has associated publications on action selection (Rohlfshagen & Bryson 2008). 

This library was first developed by Tristain Caulfield for EASSS in July 2005.  Tristan was primarily trying to connect jyPOSH (by Andy Kwong) to MASON and never really worked with or understood BOD.  It was updated for the new versions of BOD/MASON by Jan Drugowitsch in 2006-7.   It is being improved for better behavior decomposition (and better behaviour!) by Joanna Bryson in 2008.

What does it do?

There are two types of agents distributed randomly around the field --- sheep and dogs.  In Tristan's version (which is the default), the dogs just chase the sheep.  The sheep flock (see Reynolds 1987).  They also panic (see Delgado Mata & Aylett 2005), which means they move closer to each other when they either sense a predator or sense that the other sheep are panicked.  The flocking exploits some of the underlying MASON routines for flocking.

In the new version the dogs & sheep share some behaviors, which is what a behavior library is supposed to be about:  code reuse.  But they are still very different.  The dogs now flock too, but with the sheep as well as each other.  And the smarter sheep not only pull together, but also run away from the dogs, so the dogs herd them more effectively.  To see the newer, smarter behaviour, you need to use the init_smarter_beasts agent init file, instead of just the default init_agents.

The next time I feel like I have time to play with this, I'll use the latching demonstrated in latch test to make the dogs go off & look for strays.
