latchTest (MASON)


This library was created to test a new attribute of action selection inspired by the emotion & drive durative state systems in animals.  It relies on the BOD/MASON library, and it was for this that POSH/profiler.py and exp_executer.py were written.  As a consequence, both of those files are (as of this writing) possibly too dependent on BOD/MASON and may need further fixing if to be used in another context.

This library has associated publications on action selection (Rohlfshagen & Bryson 2008).

This library was written by Philipp Rohlfshagen in the Autumn of 2007, based on the earlier macque library written by Hagen Lehmann.  Both Rohlfshagen and Lehmann were employed by the University of Bath and funded by The Engineering and Physical Sciences Research Council (EPSRC), Grant GR/S79299/01 (AIBACS), ``The Impact of Durative Variable state on the Design and Control of Action Selection''; Joanna J Bryson (University of Bath) PI.

What does it do?

Most files are `standard' behaviours, similar to the other libraries. In fact, most of the work was initially based upon the library `macaques'. The most crucial differences are:

- Most behaviours inherit from the class Latch, that automatically allows a state to be latched. The constructor takes several arguments that allow to model different latches. See for more documentation below.
- The behaviour is now subclasses simply by calling the constructor. It is no longer required to pass the list
