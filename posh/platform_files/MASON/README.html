README for jyPOSH/platform_files/MASON

This directory includes MASON classes used by BOD agents running in a 2-D Environment.  These classes are implemented in Java rather than python/jython in order to provide the MASON behaviour with optimised methods to interact with the MASON environment.

Note that because jython is running in java 1.4, these files must also be compiled for java 1.4!

Making it run

The classes need to be compiled into bytecode before being used for the first time.

This can be done by using something like:

    javac -target 1.4 -source 1.4 -cp mason.jar Entity.java OrientedEntity.java Simulation2D.java Simulation2DGUI.java

or even easier:

    javac -target 1.4 -source 1.4 -cp mason.jar *.java

If you are lucky, the jyPOSH system may do this compile for you, but if it doesn't work just do it by hand.

Documentation

mason_doc.py is a documentation stub written in python to allow epydoc do generate appropriate documentation for the classes written in Java. If the Java files are changed, the mason_doc.py has to be updated to reflect the changes.

New Versions of  MASON

There is a shell script, create_mason.jar.sh, for creating new versions of the mason.jar file when George Mason releases new versions of MASON.  This should only need to be done by us (the people distributing BOD/MASON), but just in case you want to know which version of MASON we are running, see that script file.
